## 《云计算技术》期末实验大作业（分组选题列表）

### 使用说明（先读）

- **分组人数**：建议 3–5 人/组（可按班级实际调整）。
- **选题方式**：每组从下列题目中任选 1 个；同题可多组选择，但**实现细节/参数/场景必须不同**（例如不同规模、不同网络拓扑、不同镜像初始化方案、不同监控维度等）。
- **最低交付物（所有题目通用）**：
  - **可复现实验环境说明**（硬件/系统/网络/版本/镜像来源）
  - **关键命令与配置**（可截图，但必须配文字解释；建议附命令清单）
  - **结果展示**（截图/日志/输出）
  - **风险与排错**（至少 3 个问题：现象-原因-定位-解决）
  - **答辩材料**：8–12 页 PPT（建议结构见 README）

### A 组：OpenStack（IaaS 平台搭建与运维向）

#### A1. Neutron 虚拟网络实验：网络/子网/端口/网桥的创建与联通验证（推荐）

- **目标**：理解 OpenStack 虚拟网络抽象，完成网络资源全链路创建与联通验证。
- **要求（至少）**：
  - 使用 **CLI（openstack 命令）**创建：network、subnet、port，并解释参数含义
  - 至少设计 2 个网络（如外部网络 + 内部网络）与 2 个子网
  - 创建至少 2 台云主机，验证同网互通、跨网访问（按你的网络方案）
  - 给出 1 个“故障注入”与排错过程（例如安全组、路由、端口绑定等）
- **建议扩展**：对比 Dashboard 与 CLI 的操作差异；输出网络资源清单与拓扑图。

#### A2. Flavor 实例类型（规格）体系设计：面向不同业务的云主机规格模板

- **目标**：理解云主机“虚拟硬件模板（Flavor）”的作用与管理方法。
- **要求（至少）**：
  - 设计并创建不少于 5 种 flavor（不同 vCPU/RAM/Disk）
  - 结合 2 种业务场景（如 Web/数据库/缓存/CI）解释规格取舍
  - 通过创建云主机验证不同 flavor 的资源差异（输出对比）
- **建议扩展**：加入配额（quota）视角讨论“资源池化与可计量服务”。

#### A3. 云主机与快照管理：批量创建、生命周期管理与快照回滚演示

- **目标**：掌握 server 的创建/启停/删除/锁定/暂停，以及快照的用途。
- **要求（至少）**：
  - CLI 创建云主机（至少 2 台），展示生命周期管理命令及效果
  - 创建快照并用快照快速恢复/重建一台云主机
  - 给出“快照适用场景 vs 镜像适用场景”的对比结论
- **建议扩展**：编写脚本实现批量创建/批量操作（含错误处理）。

#### A4. 镜像部署与初始化：Key Pair + cloud-init + 安全组的生产化登录方案

- **目标**：理解云镜像部署、SSH 密钥对、cloud-init 初始化以及安全组策略。
- **要求（至少）**：
  - 创建并管理 key pair，完成 SSH 登录
  - 编写 cloud-init（至少完成：创建用户/写入公钥/安装 1 个服务/写入配置文件之一）
  - 设计并验证安全组规则（入站/出站至少各 1 条），解释最小权限原则
  - 部署一台 CentOS（或同类 Linux）云主机并验证初始化生效
- **建议扩展**：实现“免密码批量运维”示例（注意安全合规说明）。

### B 组：虚拟化与资源抽象（概念 + 实操/对比向）

#### B1. 虚拟机 vs 容器对比实验：资源开销、启动速度与隔离边界

- **目标**：用实验数据说明虚拟化与容器的差异。
- **要求（至少）**：
  - 选定同一应用（如 Nginx/Redis/简单 Web 服务），分别用 VM 与 Docker 运行
  - 对比：启动时间、CPU/内存/磁盘占用（至少 3 个指标）
  - 给出“隔离边界”解释：OS 级虚拟化 vs 应用层抽象
- **建议扩展**：解释与云计算特征“快速弹性伸缩”的关联。

#### B2. 资源池化与可计量服务：设计一套“按量计费”指标与采集方案（课程综合题）

- **目标**：把 NIST 云计算特征落到可观测与计量上。
- **要求（至少）**：
  - 设计可计量指标：计算/存储/网络至少各 1 个
  - 说明采集方式（命令/日志/监控组件/脚本均可）与数据格式
  - 给出一个“计费示例”：按某一计费模型计算 1 天费用
- **建议扩展**：加入 SLA 指标（可用性/延迟）并说明监控告警策略。

### C 组：Docker 与微服务（容器技术实战向）

#### C1. Docker 镜像构建与发布：把一个应用容器化并实现可复现部署

- **目标**：掌握 Dockerfile、镜像分层、容器运行参数与镜像发布流程。
- **要求（至少）**：
  - 编写 Dockerfile，构建镜像并解释关键指令（FROM/RUN/COPY/EXPOSE/CMD 等）
  - 用 `docker run` 跑起来，给出端口映射、卷挂载、环境变量的使用示例
  - 给出镜像版本管理策略（tag 命名规则）
- **建议扩展**：做一次镜像瘦身（多阶段构建/清理缓存）并对比体积变化。

#### C2. 微服务容器编排：用 Docker Compose 部署一个“前端+后端+数据库”三层应用

- **目标**：理解服务发现、网络、持久化、配置与依赖编排。
- **要求（至少）**：
  - `docker-compose.yml` 包含不少于 3 个服务
  - 数据库必须持久化（volume），并解释数据持久化验证方法
  - 给出健康检查或启动顺序控制方案（healthcheck / depends_on 说明）
- **建议扩展**：加入日志采集或简单监控（如容器指标采集）。

### D 组：Kubernetes（快速实战与核心原理向）

#### D1. K8s 部署与滚动升级：Deployment + Service + Ingress（或 NodePort）完整链路

- **目标**：掌握容器编排基本对象与滚动升级机制。
- **要求（至少）**：
  - 部署一个应用（Deployment），并配置 Service 暴露
  - 演示一次滚动升级与回滚（提供命令与输出/截图）
  - 说明副本数、探针（liveness/readiness）与可用性的关系（至少 1 个探针）
- **建议扩展**：加 ConfigMap/Secret 管理配置。

#### D2. 容器可观测：为 K8s 应用设计“监控 + 日志 + 告警”最小闭环

- **目标**：把“可计量服务/运维能力”落到集群层面。
- **要求（至少）**：
  - 至少给出监控指标（QPS/延迟/资源使用等）与采集展示方式
  - 至少给出 1 条告警规则（触发条件、阈值、处理建议）
  - 日志收集方案说明（可简化为标准输出采集 + 查询示例）
- **建议扩展**：解释容量规划思路与扩缩容策略（HPA/手动均可）。

### E 组：云原生综合（DevOps/交付链路向）

#### E1. 从代码到上线：CI 构建镜像 + 推送镜像仓库 + 部署到 K8s（或 Docker Compose）

- **目标**：体验云原生交付链路与版本可追溯。
- **要求（至少）**：
  - 设计一条流水线（可以用 GitHub Actions/本地脚本模拟）完成：构建 -> 测试（可简单）-> 打包镜像 -> 推送
  - 部署端验证：升级与回滚至少演示一次
  - 给出“版本追溯”说明：从运行实例定位到代码/镜像 tag
- **建议扩展**：引入安全扫描或依赖检查（可选）。
